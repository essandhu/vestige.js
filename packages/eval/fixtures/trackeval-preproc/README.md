# trackeval-preproc — cross-implementation fixture for the MOT17 pipeline

Oracle for the **whole eval pipeline**: MOTChallenge parsing → TrackEval-style
preprocessing (`preprocessMotSequence`) → CLEAR-MOT / Identity / HOTA metrics.
Generated by running the canonical reference, `JonathonLuiten/TrackEval`
(CONTRIBUTING.md §4.2), on a synthetic MOT17-style sequence; consumed by
`tests/validation/trackeval-preproc-fixture.test.ts`.

Conventions follow ADR-0002 (per-fixture directory, committed `gen.py` +
`data.json`, generator provenance in the JSON envelope). This is the eval
package's own fixture tree, anticipated by ADR-0002 §1; its
`../requirements.txt` pins **different versions than core's**
(`numpy==1.23.5`, `scipy==1.10.1`) because TrackEval uses the `np.int` alias
removed in numpy 1.24.

## What the sequence covers and why

12 frames, hand-specified (no RNG — regeneration is deterministic). The gt
exercises every preprocessing branch separately:

| gt id | class | consider | Purpose |
|---|---|---|---|
| 1 | 1 pedestrian | 1 | evaluated track, present all 12 frames |
| 2 | 1 pedestrian | 1 | evaluated track, frames 1–8; its tracker coverage id-switches (102 → 103) |
| 3 | 7 static_person | 1 | **distractor**: tracker id 104 follows it and must be *removed* by preprocessing |
| 4 | 1 pedestrian | **0** | zero-marked: dropped from eval gt, but tracker id 105 following it is *kept* (→ FPs) |
| 5 | 3 car | 1 | valid non-evaluated class: dropped from eval gt, tracker id 107 following it is *kept* (→ FPs) |

Rows 3/4/5 distinguish the three different "gt row disappears" behaviors —
only the distractor class removes tracker detections. Tracker id 106 is a
spurious far box (plain FPs). All boxes are well separated and offsets give
IoU ≈ 0.9 to the followed gt, so no assignment ties exist and scipy-vs-JV
tie-breaking cannot make the comparison flaky.

Hand-checkable consequences (all confirmed in `data.json`): 32 of 42 tracker
dets survive (104's 10 removed), CLEAR TP=20 FN=0 FP=12 IDSW=1 → MOTA=0.35,
IDTP=17 / IDFN=3 / IDFP=15.

## Fixture shape

- `sequence.gtLines` / `sequence.trackerLines` — the exact file lines; the TS
  test parses these, so both implementations see identical input.
- `preprocessed.frames[t].gtIds/trackerIds` — per-frame ids after
  preprocessing, in TrackEval's relabeled form (kept original ids mapped to
  `0..n-1` in ascending order). The TS test applies the same relabeling to
  its own output before comparing.
- `metrics.{hota,clear,identity}` — TrackEval's per-sequence results; HOTA
  fields carry both the 19-alpha arrays and their means.

## Regenerating

```sh
cd packages/eval/fixtures
pip install -r requirements.txt          # python >= 3.11
git clone https://github.com/JonathonLuiten/TrackEval /tmp/TrackEval
git -C /tmp/TrackEval checkout 12c8791b303e0a0b50f753af204249e622d0281a
TRACKEVAL_PATH=/tmp/TrackEval python3 trackeval-preproc/gen.py
```

Commit `gen.py` and `data.json` together, never one without the other
(ADR-0002 §4). The pinned TrackEval commit is recorded in the JSON envelope;
re-pinning to a newer TrackEval is a deliberate re-baseline.
